# Описание

# Хранение данных
Данные о посещениях сохраняются в Highload-блоке (модуль highloadblock). Выборка данных осуществляется через ORM D7.

# Фиксация посещений
Используется обработчик события OnPageStart, зарегистрированный через EventManager в файле init.php. При каждом обращении к публичной странице проверяется, не относится ли URL к служебным разделам (/bitrix/, /upload/, /ajax/). Если проверка пройдена, через ORM-метод add() создаётся новая запись в Highload-блоке.

# Обоснование
Highload-блок это отдельная таблица в БД, обеспечивает высокую производительность с большими объёмами однотипных записей, имеет встроенный административный интерфейс и легко расширяется новыми полями.
Событие OnPageStart вызывается в самом начале загрузки страницы без влияния на отрисовку контента.

# Ограничения
Не фиксируются AJAX-запросы к публичным обработчикам, если они не инициируют полную загрузку страницы.
При высоких нагрузках запись в БД на каждый хит может создавать задержки, нужно делать запись асинхронно через агенты на кроне или очереди.
IP-адрес определяется через $_SERVER['REMOTE_ADDR'], что может давать адрес прокси-сервера.
